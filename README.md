# Мышонок Мышь 🐭

Статический сайт со сказками о мышонке Мыше и его «памятью» — друзьями и вещами, которые он узнал. Построен на Astro, деплоится на GitHub Pages.

## Запуск локально

```bash
npm install
npm run dev      # http://localhost:4321
npm run build    # сборка в ./dist
```

## Деплой на GitHub Pages

1. Создайте репозиторий и запушьте проект в ветку `main`.
2. В `astro.config.mjs` укажите свои значения:
   - `site: 'https://ВАШ_ЛОГИН.github.io'`
   - `base: '/ИМЯ_РЕПОЗИТОРИЯ'` (если сайт в подпапке; для своего домена в корне — `base: '/'`).
3. В настройках репозитория: **Settings → Pages → Source → GitHub Actions**.
4. Каждый пуш в `main` автоматически собирает и публикует сайт.

## Настройки чтения

В шапке есть кнопка «Чтение»: переключение **темы (день/ночь)**, **размера шрифта** (4 ступени) и **межстрочного интервала** (3 ступени). Выбор сохраняется в браузере (`localStorage`). Тёмная тема по умолчанию следует системной.

## Как добавить новую сказку

1. **Сказка** — новый файл в `src/content/stories/`, например `02-vodomerka.md`:
   ```yaml
   ---
   number: 2
   title: "Водомерка"
   place: "озеро"
   characters: ["vodomerka"]
   things: []
   date: 2026-06-29
   ---
   (текст сказки)
   ```
2. **Новый друг** — добавить запись в `src/data/characters.yaml` (ключ = id):
   ```yaml
   vodomerka:
     name: Водомерка
     emoji: 🦟
     appearance: "…как выглядит глазами мышонка…"
     home: на глади озера
     metInStory: 2
     facts:
       - "…"
   ```
3. **Новая вещь/явление** — добавить запись в `src/data/things.yaml`.
4. В сказке перечислите `characters` и `things` по их `id` (= ключам в YAML).
5. `git commit && git push` — сайт обновится сам.

> **Память** живёт в `src/data/characters.yaml` и `src/data/things.yaml` — компактно и удобно читать/дополнять (в т.ч. агенту). Поля проверяются по схеме при сборке: опечатка или пропущенное поле остановят билд с понятной ошибкой (например, `Память (друзья) → «vodomerka»: metInStory Required`).
>
> YAML-памятка: если в строке есть `:`, `#` или кавычки — бери её в "двойные кавычки".

## Структура

- `src/content/stories/` — сказки (проза, Markdown + frontmatter).
- `src/data/characters.yaml`, `src/data/things.yaml` — «память»: друзья и вещи.
- `src/lib/memory.ts` — загрузка и валидация памяти, типизированные выборки.
- `src/content/config.ts` — схема сказок.
- `src/layouts/Base.astro` — каркас страницы.
- `src/components/ReadingControls.astro` — панель настроек чтения.
- `src/pages/` — страницы и маршруты.
- `src/styles/global.css` — токены, темы, типографика.

## История чтения (без авторизации)

Прогресс хранится локально в браузере (`localStorage`, ключ `mt-history`) — без сервера и логина:
- на странице сказки кнопка **«Отметить прочитанной»** ставит/снимает отметку;
- открытая сказка запоминается, и на главной появляется блок **«Продолжить чтение»**;
- прочитанные сказки помечаются на главной значком «прочитано»;
- в подвале есть ссылка **«Очистить историю чтения»**.

История привязана к одному браузеру/устройству. Если позже понадобится синхронизация между устройствами — это место расширяется внешним сервисом авторизации (например, Supabase) без переделки остального.
